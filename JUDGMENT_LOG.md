# 判断記録 (Judgment Log)

このドキュメントは、PdM (筒井) が AI コーディングアシスタント (Claude Code) と並走しながら本リポの仕様策定・実装を進めた際に、**「AI が出した初期案」と「PdM が rollback または修正して採用した最終案」の差分とその理由** を時系列で記録するものです。

## なぜこの記録を残すのか

AI 補助環境では output 量が増える反面、**「どこが PdM の判断で、どこが AI 由来か」が外部から見えにくくなる** 課題があります。本記録はその透明性を確保し:

- PdM 固有の付加価値 (ドメイン知識・運用判断・UX 直感) を可視化
- 同じ判断軸を別の案件でも再現できるよう参照可能化
- 採用しなかった A 案も残し、「なぜ B が良かったか」の根拠を後から検証可能に

を目的としています。

## 運用ルール

1. **仕様変更時**: Claude が初期案を出し、PdM が rollback / 修正した場合は本表に追記
2. **commit message**: 「`判断記録: rollback判断: A (旧) → B (採用), 理由: ...`」のフォーマットを採用
3. **A 案でも動いたケース**: 「結果論ではなく判定基準の根拠」を残すために A 案も記録
4. **半年に 1 回**: 本表をレビューし、本リポ固有の判断軸を末尾セクションに抽出

## 判断詳細

### J-001 — YYYY-MM-DD — テーマ

**背景** (何が課題だったか、どんな機能改修の文脈か):
- 1-2 行で

**Claude 提案 (A 案)**:
- AI が初期に提示した案を、なるべく原文に近い形で記述

**PdM 採用 (B 案)**:
- 最終的に採用した案。Claude が出した案を修正・rollback した内容を明示

**採用理由**:
- なぜ B 案を選んだか。実運用上の懸念・ドメイン知識・UX 原則などを根拠付きで

**効果 / 振り返り** (オプション):
- 採用後にユーザーや顧客から得られたフィードバック、改修必要性など

## 本リポで一貫して優先した判断軸 (継続更新)

本リポでの判断記録が積み重なってきたら、以下のような形で **再現可能な原則** として抽出していきます。半年〜1 年ごとに見直し:

例:
- 判別不可ケースで UI が破綻しない設計を優先
- 責務境界 (どこからどこまでがこの機能の領分か) の明確化
- 冗長を削る (UI 表記、説明文、機能オプション)
- ブラウザネイティブ UI を排除しカスタム UI に統一
- 明示的なユーザーアクション (自動遷移より「次へ」明示クリック)
- 状態管理の排他制御 (checkbox より radio をブラウザ強制で活用)
- AI の「自分の構造に従いがち」傾向を見抜くメタ判断
